### 버전관리
- 안정버전(기본값): curl -fsSL https://claude.ai/install.sh | bash
- 최신 기능: curl -fsSL https://claude.ai/install.sh | bash -s latest
- 특정 버전: curl -fsSL https://claude.ai/install.sh | bash -s 2.0.75

### 설정 디렉토리
```
project-root/
├── .cluade/
│   ├── commands/             # 커스텀 슬래시 명령어
│   ├── agents/               # 커스텀 에이전트 정의
│   ├── skills/               # 에이전트 스킬 모듈
│   ├── hooks/                # 이벤트 훅 스크립트
│   ├── output-styles/        # 출력 스타일 템플릿
│   └── setting.json          # 프로젝트별 클로드 코드 설정
└── CLAUDE.md                 # 프로젝트 지침서(컨텍스트로 자동 로드)
```

### 실행방법
```
- 일회성 작업: claude "질문"
- 비대화형 쿼리: claude -p
- 최근대화 세션 재개: claude -c
- 이전 대화 세션 중 선택 재개: claude -r
```

### 명령어
- /clear: 현재 대화 컨텍스트 초기화. 프로젝트 설정과 체크포인트 유지 되므로 /rewind로 이전 상태 복구 가능
- claude doctor: 정상 설치 여부 확인. 설치 유형, 경로, 상태 등을 진단하여 출력

### checkpointing
- 파일 편집을 자동으로 추적하므로 빠르게 실행 취고하소 문제가 발생한 경우 이전 상태로 되돌릴 수 있음.
- `/rewind` 실행하거나 프롬프트 입력이 비어 있을 때 `Esc`를 두 번 눌러 rewind 메뉴를 연다.
- 프롬프트가 있을 때 Esc를 두번 누르면 메뉴를 열지 않고 텍스트를 지움
```
#rewind 메뉴
- 코드 및 대화 복원: 코드와 대화를 해당 지점으로 되돌림
- 대화복원: 현재 코드를 유지하면서 해당 메세지로 되돌림
- 코드 복원: 대화를 유지하면서 파일 변경 사항을 되돌림
- 여기서부터 요약
- 여기까지 요약
- 취소

```
- 유용한 상황
1. 시작점을 잃지 ㅇ나혹 대체 구현을 탐색해야 할 때
2. 잘못된 변경으로 버그가 생겼거나 기능이 깨져서 빠르게 복구해야 할 떄
3. 작동하는 현 상태를 유지하면서 실험해야 할 때
- 한계
```
1. Bash를 통한 파일 변경은 추적하지 못하므로, 전용 편집 도구를 사용하도록 유도하는 편이 안전
2. 외부 변경사항은 기록되지 않음. 별도 편집기에서 직접 수정, 다른 세션 등
3. 버전 관리 시스템의 대체재가 아님.

```
